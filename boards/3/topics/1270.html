[CLI]: Useful (if long) one-liner for writing from sidecars to files

Added by Steve Wright about 9 years ago

Cygwin users (like myself) take note: except for the one call to Exiv2, this whole line is pure BASH.

for file in *.exv; do target=${file/exv/jpg} ; if [ -f "$target" ]; then exiv2 -vkia "$target"; fi; done

You could even -- and I just may, for my own purposes -- write a script with just the above and call it "writeback.sh" or something. G

I thought I'd share this now, before I forgot it or became too used to using it.

For Windows/cmd.exe users, this can be done with the following (horrible) command:

for %F in (*.exv) do if exist %~dF%~pF%~nF.jpg exiv2 -vkia %~dF%~pF%~nF.jpg
I suggest putting this into a .bat file with @echo off on the first line. And to make sure it's cryptic, double the %%'s. So, writeback.bat
@echo off
for %%F in (*.exv) do if exist %%~dF%%~pF%%~nF.jpg exiv2 -vkia %%~dF%%~pF%%~nF.jpg

for %F in (*.exv) do cmd ... is obvious and makes sense
if exist ... is obvious and makes sense

%~dF = drive for %%F, p = path, n = name
So %~dF%~pF%~nF.jpg replaces .exv with .jpg

So this command says:
for every F in (*.exv) do if exists (replace .exv with .jpg) exiv2 -vkia (replace ...)

It's easy to dislike cmd.exe's batch script syntax.
