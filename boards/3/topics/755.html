RE: set location though exiv2 ? - Added by Andreas Huggel almost 11 years ago

Julien,

Quick answer/workaround: Just leave the type specifier ("String") out of your commands and you should be fine.

Long answer: The type "String" is only suitable for IPTC tags. XMP properties require XMP specific types; the type for XMP properties with simple string values is "XmpText". So if you replace "String" in the commands that set XMP properties with "XmpText" the commands will work.
In addition, Exiv2 has a default type for all known tags. That default type is used when no type is explicitly given in a command like yours. The default type for Iptc.Application2.Credit is "String" and for the two XMP properties it is "XmpText".
As for the assertion that this triggers, I'll have to replace it with a proper exception and error description. Looks like I couldn't imagine a situation that would trigger this assertion when I coded it. (The assertion is better than nothing though.)
Robin, the Todo there is about a related design issue: If Xmpdatum had an XmpValue instead of a Value member, the dynamic cast wouldn't be necessary at all at this place. Somehow that wasn't straightforward to achieve, I don't remember exactly why offhand.
