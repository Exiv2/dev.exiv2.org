RE: Account creation - Added by Andreas Huggel over 12 years ago

Hi Marijn,

I only get 4 four warnings like this:

Warning: Directory Pentax, entry 0x0000 has unknown Exif (TIFF) type 0; setting type size 1.

That means Exiv2 doesn't know how to interpret these tags in the Makernote, as they have an unknown type value. Therefore it doesn't know how many data bytes to read and assumes each component is 1 byte long. But since the number of components for these tags is 0, it doesn't really matter.

In short: you can safely ignore these warnings

Andreas

RE: Account creation - Added by Andreas Huggel over 12 years ago

Hi Marijn,

Thanks for testing. TypeInfo::typeName returns 0 if the type is unknown. I hesitate to change this behaviour, it would (a) come close to an API change and (b) make it more difficult for applications to check for valid types, e.g., like the exiv2 utility currently does. I've changed the doc and samples/exifprint.cpp instead, see r1757 and r1758

Andreas
