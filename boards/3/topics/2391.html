RE: Error: Directory (Last IFD item) with 26009 entries considered invalid; not read. - Added by Robin Mills over 5 years ago

I believe this message is generated when the Exif metadata references data outside of the current "block/segment of the file". This is forbidden by the spec, however some manufacturers (Sony for sure) frequently store preview images in another segment of the file.

You might find the options -pS and -pR useful to help understand the structure of your file. -pS was added in v0.25 and -pR is still work-in-progress on the trunk. There are documents in the Wiki which discuss the structure of TIFF/JPEG/PNG files. I believe CRW is a TIFF style file format.

I don't think the LensType has anything to do with this message. Canon frequently use LensType = 65535 0xffff = -1 to say "I don't recognise this lens type". Exiv2 has code to inspect other metadata (such as focal length and aperture) to guess the lens type. So the same lens might be recognised on one camera body, but not on a different camera.

Warnings are normally written to errout, so your script can ignore the warning by running exiv2 ..options.. image.crw 2>/dev/null
