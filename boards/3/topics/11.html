[BUG] Reading Metatags in PNG image

Added by Anonymous almost 13 years ago

Hi,

I found some problem which is there in the stable 0.18 win build from the official webpage, which was not there in an (unofficial) KDE win build with version 0.17. patch 91 (according to version.hpp). I didn't found any information about a similar bug, so I report it here:

- Input image: normal PNG file (without ANY metadata: no exif, no iptc)
- executing "exiv2.exe pa image.png" crashes the program (no output is displayed)
 executing "exiv2.exe ps images.png" displays basic summary information as expected
 since exiv2 is (could be) used as a dll, the whole program (not just exiv) crashes in this case...

Thanks in advance,
Daniel

Hi Daniel,

It doesn't crash here. If the file doesn't contain any metadata, exiv2 -pa simply outputs nothing. Try exiv2 -v -pa image.png to get more verbose output, it should print warnings.
