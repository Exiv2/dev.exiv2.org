[Maybe bug]: Pentax-DA 18-135mm recognition

Added by Dom Som over 4 years ago

I am using Darktable 2.2.5 on Ubuntu 16.04, exiv2 0.25-2.1ubuntu16.04.1pmjdebruijn1~xenial (darktable ppa), lensfun 0.2.8-3pmjdebruijn2~xenial (darktable ppa).
The issue is that Darktable don't recognize my camera lens in the pictures. The name between lensfun and exiv2 is slightly different:

@$ dpkg -L liblensfun-data | grep 'Pentax-DA 18-135' /usr/share/lensfun/*.xml
/usr/share/lensfun/slr-pentax.xml: <model>smc Pentax-DA 18-135mm f/3.5-5.6 ED AL IF DC WR</model>

$ exiv2 -pt IMGP6606.DNG | grep 'PENTAX-DA'
Exif.PentaxDng.LensType Byte 4 smc PENTAX-DA 18-135mm F3.5-5.6 ED AL [IF] DC WR@

On Lensfun's faq they said:
"If the lens name is not correct in the above output please report a bug at the exiv2 issue tracker"

So...here I am...
Can you/I fix it?
Thanks
Dom

There is a new feature in Exiv2 v0.26 which enables you to fix it. This is discussed here: http://dev.exiv2.org/boards/3/topics/2854?r=2861#message-2861

Exiv2 v0.26 was released in April 2017. I don't know the schedule for Darktable providing support for Exiv2 v0.26. I believe Darktable will expect Exiv2 v0.26 to be installed as part of Linux.
