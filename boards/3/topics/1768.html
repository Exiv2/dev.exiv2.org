RE: Use for a custom board - Added by Robin Mills over 7 years ago

We support Autotools (./configure etc) and CMake build environments. Both support cross compiling. Additionally we support Visual Studio for Windows and that is Microsoft proprietary build magic.

Most boards don't support make, gcc and other development tools. However, you should be able to cross compile for your board if you have the tools for your board installed on Linux. Here's an article I googled about cross compiling with Autotools: http://www.gnu.org/software/automake/manual/html_node/Cross_002dCompilation.html

If you're working on Windows, I've found Eclipse to be a very pleasant development environment and configure my projects to build on Linux over ssh. I don't think I've ever had to debug on the board from Eclipse on Windows, however I believe that's possible.

I've worked on various embedded systems, however I've only ever built Exiv2 for desktop and servers. I don't recall this being discussed on the Forum, so you could be trailblazing here.

I feel that cross compiling is a platform issue and not really something for Exiv2. So, Google is probably you best friend here. Our standard release process for Exiv2 uses cross compilation to build Cygwin (Windows) executables on a Linux machine. If you're stuck, please let me know and I'll point you to our release code and that might help.

RE: Use for a custom board - Added by Robin Mills over 7 years ago

Yes, I used Code Sourcery on a project 3 years ago. Very good product. I debugged the board from CS (Eclipse) running on Windows. We had a JTAG cable/yellow adapter from the PC's USB to the serial port on the board. It worked very well both for loading code and debugging. On that project there was no operating system. STL provide an implementation of the Std "C" library and some magic to handle interrupts and stuff.

You can go down the road of Rules.make, however I think you'll be able to build Exiv2 without modifying config/config.mk. Run ./configure --target to generate your Makefile in a terminal. In CS (Eclipse), you can open a "New Project / With existing makefile". Make and GCC will run on your desktop and cross compile for your board. You should be able to push the binary onto the board over the cable.
