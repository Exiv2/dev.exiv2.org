Newbie help sought please - not working since Win7 reinstall?

Added by Anonymous Poster over 6 years ago

Hi there

I'm not a coder, but would be grateful if you can consider my plight? I managed to figure out how to use Exiv2.exe to push IPTC into tiff files as part of my workflow. It's worked fine for several years, until last week when I had to reinstall windows 7 64 onto my main drive.

Since then I have been unable to get things to run at all - even with it on a secondary HDD? The screen just says: " Exiv exception in print action for file.... the file contains data of an unknown image type" and closes (even without a file to the corresponding line in the .bat file). I haven't changed anything? I did notice that in the file properties there is now a security box saying 'This file came from another computer and might be blocked to help protect this computer'. I've clicked unblock on this for exvi2.exe and libexpat.dll but no luck. The icons that used to be on my .bat files (meshing cogs) has been replaced with generic missing icon type? There aren't that many moving parts, but it has be stumped?!

Strangely, I can run the .bat files over my network from another computer?

I'm assuming the new install on Win7 has done something to the permissions? Your erudite advice would be gratefully received. Kind regards, Rob

Here's a line from the bat file - which has worked fine for years.

RE: Newbie help sought please - not working since Win7 reinstall? SOLVED - Added by Anonymous Poster over 6 years ago

Solved it myself. Found that file associations have become mucked up with the Win 7 reinstall. Used this utility (http://www.winhelponline.com/blog/unassociate-file-types-windows-7-vista/) to remove the association .bat had formed with EXIV2. Works fine now. That only took days.....
