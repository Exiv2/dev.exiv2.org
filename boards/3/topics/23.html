Compilation problems in Windows XP + MinGW

Added by Anonymous almost 13 years ago

Hi!

I had compilation problems when compiling exiv2 (0.18) on my WindowsXP machine using minGW and MSYS. Creating the library libexiv2.a seems to go well, but as soon as I try to use one of the functions of the library, the linker complains about missing references :
taglist.o(.text+0x1a8):taglist.cpp: undefined reference to `_imp___ZN5Exiv28ExifTags7taglistERSo'
taglist.o(.text+0x20f):taglist.cpp: undefined reference to `_imp___ZN5Exiv28ExifTags7taglistERSo'
taglist.o(.text+0x378):taglist.cpp: undefined reference to `_imp___ZN5Exiv212IptcDataSets11dataSetListERSo'
taglist.o(.text+0x3db):taglist.cpp: undefined reference to `_imp___ZN5Exiv28ExifTags14ifdIdByIfdItemERKSs'
taglist.o(.text+0x3ec):taglist.cpp: undefined reference to `_imp___ZN5Exiv28ExifTags10isMakerIfdENS_5IfdIdE'
taglist.o(.text+0x408):taglist.cpp: undefined reference to `_imp___ZN5Exiv28ExifTags7taglistERSoNS_5IfdIdE'
taglist.o(.text+0x4a8):taglist.cpp: undefined reference to `_imp___ZN5Exiv213XmpProperties15printPropertiesERSoRKSs'

Debugging this problem, I noticed that while linking libexiv2, the linker warns that :
   libtool: link: warning: undefined symbols not allowed in i686-pc-mingw32 shared libraries
Googling this, I found that they had similar problems in other projects, and solution was to link with -no-undefined flag. It worked for exiv2 too! On my set-up, I need to run configure with the following parameters:

./configure --without-zlib --disable-xmp LDFLAGS=-no-undefined

I am completely novice with exiv2 and also with Win32 environment : I don't understand why no-undefined fixes my problem. I would be thankful if someone could brief me on what's happening here... In any case, the main goal of my message is to tell how I fixed the problem, in the unlikely case somebody else faces the same issue.

Kind salutations to the team!
-Bert
