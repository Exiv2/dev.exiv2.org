RE: 'Exif.Photo.UserComment' getting cropped in 3 Windows GUI applications. - Added by Andreas Huggel almost 10 years ago

when writing EXIF tags, I always include "Ascii" in the command

The type for the Exif.Photo.UserComment tag to be written correctly is "Comment" (or nothing - Exiv2 will determine the default type of the tag if you don't explicitly provide one). If you request an "Ascii" type, the comment-specific charset info will not be added. Exif.Photo.UserComment is the only tag that has this type. Your use of "Ascii" instead explains the issue you describe, I can reproduce this here. Generally, I suggest you don't provide a type and let Exiv2 determine it. See the manpage for the syntax of the "Comment" type, if you want to specify a charset. (And btw, the reference pages do list "Comment" as the type for this tag.)

Andreas
