Picture Management Question

Added by Mick Sulley almost 9 years ago

Hi, I am new to all this so please bear with me.

I have tried to find a picture management system that allows me to record who is in each picture, location, event, etc and then to get a list of all pictures of a specific person, or location, or event, or combination. As I cannot find anything that comes close I am trying to write something. As a basis my current thoughts are to add tags for people, location and event (maybe other criteria as well) so that this is stored as part of the picture file, then read this into a MySQL database for running queries to access the pictures.

Questions -
Has anyone done this? Seems likely but I have found nothing.
Which tags should I use, exif, XMP, something else?
What format should I use for the tags to future proof the design as far as possible.
Currently planning to use Python and MySQL on Debian, any reason why something else would be better?

Thanks
Mick

RE: Picture Management Question - Added by Robin Mills almost 9 years ago

Mick

Exiv2 is technology to enable the kind of application you have described. There are numerous picture management systems. The open source project "DigiKam" does many things and uses exiv2. So before you start to write code, have a look at that. I believe digikam uses sqlite as a database, however I suspect you can change the database engine.

It's very tempting to say "Oh, I want to have my own system, then it'll do exactly what I want." Try to resist that temptation. If you find an open-source project that's close to your ideas, please consider contributing. By working together, we all get further along the road with shared effort.

I've used the pyexiv2 wrapper to call exiv2 from python. It works well. On Ubuntu, you can install it with '$ sudo apt-get install pyexiv2'.

Robin
