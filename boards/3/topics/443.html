RE: Using exiv2 lib in Borland C++ IDE - Added by Robin Mills over 11 years ago

I'm sure it's possible to build exiv2 with Borland C++, however we don't support it.

We support the various UNIX systems (including the Mac and Winodws/Cygwin) with the usual autoconf procedure (make config ; ./configure; make etc...) and DevStudio. It was quite a lot of work to create the build environment for DevStudio for the exiv2 library and application and all the sample applications. We support both static and dynamic (DLL) linking.

The simplest way to work with the Borland C++ is probably to build (or download pre-built binaries of) the Cygwin libraries and link them into your application. I'd suggest you might want to try to create a command-line application with Borland C++ and use the exifprint.cpp sample application source code. If you're can successfully compile and link that with the Cygwin libraries then you're well on your way to success.

It's not a waste of time to try this - however I don't believe anybody on the exiv2 team can offer your any assistance with this task. If you are successful, perhaps you could update this thread for the benefit of others wishing to use exiv2 and Borland C++.
