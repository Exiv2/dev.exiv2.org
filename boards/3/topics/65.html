embedding metadata into TIFF and PNG

Added by Anonymous almost 13 years ago

I have a FireFox extension called met.a [www.meta.projectmio.com] (FF2 required) which will embed relevant data into an image's metadata (JPEG) when downloaded. Data like the Title of the webpage, the URL and any portion of selected text. In the new revision I would like to add support for more formats, typically TIFF, PNG and GIF ~although GIF seems like a no go.

The problem I have is that I cannot be sure that those formats have the flexibility that JPEG has, met.a is to work on Mac OS X, Window Vista and Linux, using both IPTC and XMP standards. How I work it is that when on use in OS X and Linux images are tagged with IPTC fields and when in Windows it uses XMP fields - all viewable, editable and searchable in their respective environments, natively.

I need to know if TIFF and PNG can be tagged with IPTC and XMP, and to the following fields which met.a currently uses:

IPTC Headline: title of webpage
IPTC Caption: portion of text selected
IPTC Subject Reference: URL of webpage

XMP Title: title of webpage
XMP Comments: portion of text selected
XMP Subject: URL of webpage

I also need to know if there are any char restrictions in either of the formats?

I have sort of given up on GIF and embedding data doesn't look like its going to work, let me know if otherwise.

I have included an image ("NagueshiTemple6.jpg") which was downloaded on a Mac (IPTC) and so you can see how it looks from my end.

many thanks for any help with this.
