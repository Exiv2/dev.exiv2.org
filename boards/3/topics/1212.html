Compile Universal Binary for Mac OS X?

Added by Avk Aplmkt over 9 years ago

Hello,

I am trying to compile exiv2 into a statically-linked Universal Binary that runs on both PowerPC (PPC) and Intel Macs. Using the tips here1, I exported the following environment variables before running ./configure and make:

export SDK="-isysroot /Developer/SDKs/MacOSX10.5.sdk"
export SDKLIB="-Wl,-syslibroot,/Developer/SDKs/MacOSX10.5.sdk"
export MACOSX_DEPLOYMENT_TARGET="10.5"
export ARCH="-arch i386 -arch x86_64 -arch ppc -arch ppc64"
export CFLAGS="-O2 -fPIC $ARCH $SDK"
export CXXFLAGS="-O2 -fPIC $ARCH $SDK"
export LDFLAGS="$ARCH $SDKLIB"
export CC="/usr/bin/gcc-4.2"
export CXX="/usr/bin/g++-4.2"
export OBJC="/usr/bin/gcc-4.2"

Everything seemed to work until I try to run make, which said:

if test "x1" = "x1"; then cd xmpsdk/src && make xmpsdk; fi;
libtool: compile: g++ -O2 -fPIC -arch i386 -arch x86_64 -arch ppc -arch ppc64 -isysroot /Developer/SDKs/MacOSX10.5.sdk -fvisibility=hidden -fvisibility-inlines-hidden -Wall -Wcast-align -Wpointer-arith -Wformat-security -Wmissing-format-attribute -Woverloaded-virtual -W -MMD -I. -DEXV_LOCALEDIR=\"/usr/local/share/locale\" -I../include -funsigned-char -DNDEBUG=1 -DEXV_HAVE_STDINT_H=1 -c XML_Node.cpp -fno-common -DPIC -o .libs/XML_Node.o
g++-4.2: -E, -S, -save-temps and -M options are not allowed with multiple -arch flags
make[1 ]: * [XML_Node.o] Error 1
make: * [xmpsdk] Error 2

Looks like the key problem is "g++-4.2: -E, -S, -save-temps and -M options are not allowed with multiple -arch flags". Does anyone have experience or tips on how to solve this problem, and more easily compile Mac OS X Universal Binaries?

Thank you very much!

RE: Compile Universal Binary for Mac OS X? - Added by Robin Mills over 9 years ago

There's a script contrib/buildForMac which will build a universal. I built shared i386/x86_64/ppc libraries for Phatch a couple of years ago. I believe you can get the script to build static ppc/x86_64. You'll have to edit the script to respect ppc:

for arch in '-arch i386' '-arch x86_64' ; do # extend the architectures

You'll see that the script builds the architectures one at a time and uses lipo to combine them into a universal.

Robin

RE: Compile Universal Binary for Mac OS X? - Added by Avk Aplmkt over 9 years ago

Actually I just got it to work right before I saw your message!

I used the "--disable-shared --enable-static --disable-dependency-tracking" ./configure flags, and the "-Bstatic" linker flag to produce an Universal (i386,x86_64,PPC) Binary that is statically linked.

I will also try the script you suggested the next time I compile exiv2.
