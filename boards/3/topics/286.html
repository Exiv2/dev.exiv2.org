Hi John,

I have installed the Windows vers into "C:\Program Files\Exiv2" but all my images are in other folders within My Pictures...Is it typical that people would keep their images in same folder as Exiv2?

The location of the Exiv2 binary doesn't matter. As long as it is in your path, Windows will find it. (To set this on my WinXP, I right-click on the "MyComputer" icon on the desktop, choose "Properties", the "Advanced" tab and click "Environment Variables". Then I have a PATH variable in both, the "User variables" as well as the "System variables" sections. Edit one of them to include the directory where the Exiv2 binary is installed. In a new cmd shell you should now be able to run exiv2 from anywhere.)
If your current working directory is not the same as the directory that contains the images, you'll need to specify them with the path on the command line.

I see there is a way to point to another folder but could you provide an example showing how using the command line, as it seems this may the only option available...

The ZIP file that you can download from the website contains an HTML user manual with several examples (at the end of the document). There is also one for the use of the -l option which you seem to be referring to:

exiv2 insert -l/tmp -S.CRW /data/*.JPG

Copy all metadata from CRW files in the /tmp directory to JPG files with corresponding basenames in the /data directory.
Note that this copies metadata as is, without any modifications to adapt it to the requirements of the target format.
Some tags copied like this may not make sense in the target image.

Whether you need that option depends on what you're trying to do. It is only used with extract and insert commands.

I did see a post describing "organize" so guess that is alternative?

The organize tool serves a different purpose. You haven't really said what you are trying to achieve so it is difficult to advise which tool to use.

Andreas

John

exiv2 is technology. Sure the sample applications are command-line programs as those can be used on windows/mac/unix - kind of everywhere really. However exiv2 isn't an application as such - it's a library from which to create applications. Having said that, the exiv2.exe application is very useful in its own right. And yes, it is a command-line program.

There are several/many open-source applications which use exiv2 and provide a GUI - for example Phatch, DigiKam and jBrout.

As Andreas said you haven't really explained what you're looking for. There will probably be a GUI program available to meet your needs which uses the exiv2 technology to do the work.
