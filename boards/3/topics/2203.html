RE: Strange behavior on Linux - Added by Robin Mills about 6 years ago

Mikayel

To build xmpsdk, you need expat to be installed on your system. Try the following command:

584 rmills@rmillsmbp:~/Downloads/exiv2-0.24 $ exiv2 -vV | grep -e expat -e exiv
exiv2 0.24 001800 (64 bit build)
exiv2=0.24.0
executable=/usr/local/bin/exiv2
library=/usr/local/lib/libexiv2.13.dylib
library=/usr/local/lib/libexpat.1.dylib
585 rmills@rmillsmbp:~/Downloads/exiv2-0.24 $
When you build, the ./configure output should appear as follows.
-- Exiv2 0.24 feature configuration summary
--
-- Build a shared library......... YES
-- Use symbol visibility support.. YES
-- PNG image support.............. YES
-- Native language support........ YES
-- Nikon lens database............ YES
-- XMP metadata support........... YES
If XMP metadata is "NO", you will have to install libexpat-devel (with apt-get or yum or something). Personally, I always download the expat source code from source-forge and build/install to the default location (/usr/local).

Once you have expat on your machine, totally reconfigure and rebuild exiv2.

$ sudo make distclean
$ make config
$ ./configure
If XMP is YES, build and test.
$ make ; sudo make install ; make samples ; make tests
If XMP metadata support is NO, or the tests fail, please send me config.status and config.log and the output from make tests.

RE: Strange behavior on Linux - Added by Robin Mills about 6 years ago

You can get the code for expat and zlib (for png support):

expat-2.1.0 http://sourceforge.net/sourceforge/expat/expat-2.1.0.tar.gz
zlib-1.2.8 http://zlib.net/zlib-1.2.8.tar.gz

I recommend that you get the code for expat and zlib and build/install it for yourself.

Different linux distros have different utilities for downloading development packages. Mac has MacPorts and brew (sudo port install expat), Ubunutu uses Synaptic (sudo apt-get install libexpat-dev), Fedora uses yum (sudo yum install expat-devel). Please use google to determine what's required for your Linux.
