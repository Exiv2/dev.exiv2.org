RE: write makernotes - Added by Andreas Huggel almost 13 years ago

Hi Marius,

Yes, it can be done. Just a few things to note:

  1. The Exif.Image.Make tag needs to be set. Exiv2 uses it to determine the type of makernote. See http://www.exiv2.org/makernote.html for the appropriate values
  2. Only one type of makernote can exist in one image or Exif block.
  3. Besides, the usual rules to set Exif tags apply

For example, using the exiv2 command line tool and a JPEG image that doesn't have any Exif data yet:

ahuggel@mowgli> exiv2 -M'set Exif.Image.Make FUJI' -M'set Exif.Fujifilm.Quality Best' exiv2-empty.jpg 
ahuggel@mowgli> exiv2 -pa exiv2-empty.jpg 
Exif.Image.Make                              Ascii       5  FUJI
Exif.Image.ExifTag                           Long        1  44
Exif.Photo.MakerNote                         Undefined  36  70 85 74 73 70 73 76 77 12 0 0 0 1 0 0 16 2 0 5 0 0 0 30 0 0 0 0 0 0 0 66 101 115 116 0 0 
Exif.MakerNote.Offset                        Long        1  62
Exif.MakerNote.ByteOrder                     Ascii       3  II
Exif.Fujifilm.Quality                        Ascii       5  Best

For C++ code, have a look at samples/write2-test.cpp

If you really need to write your own makernote class, I suggest you use one with a simple header and relative offsets, like the second Olympus makernote ("OLYMPUS\0II").

Andreas
