Dates and Times are actually stored in EXIF, listing them under IPTC is an error on the part of ACDSee.

For correct handling of Date/Time fields, I recommend reading the MWG Guidelines: http://www.metadataworkinggroup.org/pdf/mwg_guidance.pdf#page=37

As I said previously, Digikam is my recommended photo management application. Bonus is that it will see your ACDSee Categories and import them into industry standard XMP hierarchical keywords. You can read about those in the MWG guidelines as well.

RE: Is there a master list that tells what photo editor/manager software reads and stores IPTC Plus and Extension? - Added by Robin Mills over 5 years ago

I haven't really understood this thread. I'm the build engineer for Exiv2 and don't know much about metadata. As Alan has said, we're not a metadata forum. Our focus is to provide a library for applications to reliably manipulate metadata. The application exiv2(.exe) provides command-line access to most of the features of the library and is used by the test suite. If you like command-line tools, you will find exiv2(.exe) useful in metadata investigations.

I think you believe you are loosing metadata. You might find the following command very useful to extract and preserve your metadata: $ exiv2 -ea image-path. For example:

1109 rmills@rmillsmbp:~/temp $ ls -alt Stonehenge.jpg 
-rw-r--r--+ 1 rmills  staff  6757827 15 Mar 20:42 Stonehenge.jpg
1110 rmills@rmillsmbp:~/temp $ exiv2 -ea Stonehenge.jpg 
1111 rmills@rmillsmbp:~/temp $ ls -alt Sto*
-rw-r--r--+ 1 rmills  staff    17989 16 Mar 15:51 Stonehenge.exv
-rw-r--r--+ 1 rmills  staff  6757827 15 Mar 20:42 Stonehenge.jpg
1112 rmills@rmillsmbp:~/temp $
The metadata has been extracted into Stonehenge.exv. It can be restored into the image with the command: $ exiv2 insert Stonehenge.exv Stonehenge.jpg. You can list the IPTC data in an image with the command $ exiv2 -pi image. For example:
1124 rmills@rmillsmbp:~/temp $ exiv2 -pi Stonehenge.jpg 
Iptc.Envelope.ModelVersion                   Short       1  4
Iptc.Envelope.CharacterSet                   String      3  G
Iptc.Application2.RecordVersion              Short       1  4
Iptc.Application2.Caption                    String     12  Classic View
1125 rmills@rmillsmbp:~/temp $
You can remove the IPTC data with the command: $ exiv2 -di image.

I hope that helps you drill down into your issues.

RE: Is there a master list that tells what photo editor/manager software reads and stores IPTC Plus and Extension? - Added by Robin Mills over 5 years ago

As some wit once said "I love Standards. There are so many from which to choose!" It's very likely that there is metadata duplicated in the three major Standards: Exif, IPTC and XMP. And different applications will respect tags in the different Standards. A confusing mess? Yes, I'd say so.

There was a discussion about 18 months ago concerning the "Title" of an Image. Lots of applications, such as Picasa, provide a UI to give an image a "Title". I tried Lightroom, Preview.app (Mac), Picasa and some others. There was little consistency. Some stored "Title" in IPTC, some in Exif. I was pleased to see, because I'm a retired Adobe Engineer, that Adobe applications were consistent. So Lightroom, PhotoShop and Bridge all store "Title" in the same way. http://dev.exiv2.org/boards/3/topics/2016?r=2017#message-2017

The Exiv2 project tries to keep out of this jungle. We read and write metadata in a reliable manner and leave the application with the responsibility for the meaning and appropriate use of the metadata.

RE: Is there a master list that tells what photo editor/manager software reads and stores IPTC Plus and Extension? - Added by Gilles Caulier over 5 years ago

Hi Cathy,

About digiKam and ACDSee categories question, well yes, digiKam is able to import ACDSee categories as Tags in database, and may other in fact.

We take a care about interoperability with other photo management softwares, when it's possible, as LightRoom, Aperture, Nikon NX, etc..

With current stable 4.14.0, ACDSee metadata are imported as well if info are found, following and hard-coded order.

With next 5.0.0, this is fully configurable in setup/metadata config panel. End users can set the order to import/export image metadata from/to digiKam database.

https://www.flickr.com/photos/digikam/23393065632/in/dateposted-public/

Also, digiKam 5.0.0 as a new mechanism to delay when you want the sync between database and image metadata. As metadata writing in image take a while, even if multicore CPU is used,
you can process changes about image properties in digiKam, as Rating, comments, Tags, Labels, and put your changes in a queue, which will be processed later, when you take a coffre, or dinner for ex.

As you use ACDSee, i suppose that you use Windows. DigiKam is order of priority coded and tested under Linux, and after OSX, and finally Windows, when time permit. In fact, Windows porting is a very big puzzle, very more complex than Linux or OSX.
With 5.0.0, application have been ported to Qt 5.0 framework, which will simplify a lots the port and reduce the puzzle. We hope to provide quickly a Windows version to test.

RE: Is there a master list that tells what photo editor/manager software reads and stores IPTC Plus and Extension? - Added by Gilles Caulier over 5 years ago

Cathy,

M$ = Microsoft (:=)))

Gimp is not digiKam. Gimp is an advanced photo editing software, like Photoshop. I know well, as i teach photo editing for Webmaster lessons in a computer science engineer school in France.

digiKam is a photo-management software, as Aperture under OSX, or ACDSee under Windows. It's a middle and transverse software as it can edit a little bit photo (no layers support), process RAW files, edit metadata, export photo on the web services, process Panorama, and of course have a lots of search tools to find items in collection (keywords, colors, tags, quality, map, date, time-line, properties, etc...). It have also a light table to compare images side by side, a Batch queue manager to process image sets in your workflow, and a camera interface to select items from your removable devices and pre-process while downloading on you computer.
digiKam supports also virtual albums, based on tags, rating, picks and color labels.

Currently, the best database support with digiKam is local one (SQlite based), but we working on Mysql support to be able to use remote database server in case of photo agency context.

Darktable is a Raw processing software, as LightRoom is. Never tested in-deep as i found the graphical interface to much far than end users needs.

Wikipedia has a good comparison page : https://en.wikipedia.org/wiki/Comparison_of_image_viewers
