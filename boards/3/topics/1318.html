RE: How to create new VS10 project that could use exiv2 - Added by Robin Mills almost 9 years ago

It sounds has though you don't have the directory <exiv2dir>/msvc64/include on your include file search path.

During the build, the header files are copied from <exiv2dir>/src to <exiv2dir>/msvc64/include/exiv2 in order to make #include <exiv2/exiv2.hpp> work! The file exv_msvc.h should also be in <exiv2dir>/msvc64/include. The file exv_msvc.h can be changed by the user to define some aspects of the build. You'll seldom need/want to change it. However, as you've discovered, it is needed!

The sample apps (in exiv2/samples) such as exifprint are useful 'templates' for building user applications.

If you're still stuck, let me know and I'll construct a new project outside our build tree and attach it to this forum topic.

Robin

There are 4 Configurations and two platforms

Configuration/Target Win32       x64
Debug             1) Compiled with /MTd  links exiv2sd.lib (static lib)
Release           1)               /MT          exiv2s.lib  (static lib)
DebugDLL          2)               /MDd         exiv2d.lib  (stub lib)
ReleaseDLL        2)               /MD          exiv2.lib   (stub lib)

1) No post build step is required.
2) Copy dlls from \gnu\exiv2\msvc64\exiv2lib\

Recommended use:

1) Reproduce/rebuild/retest the build (even create \robin\projects)
