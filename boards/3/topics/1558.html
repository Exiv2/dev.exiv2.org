RE: GSOC 2013 - Video metadata support improvements - Added by Abhinav Badola over 8 years ago

Hi Max,

Can I know why this project has been categorized under "high" difficulty? And, can the video metadata support added last year make the addition of support for formats like MPEG and VOB somewhat easier?

I worked on Video Metadata last year as my GSoC project.
The difficulty has been set to high, because to make this project work, you have to decode a lot of binary data. Most of the time the documentation is available and it becomes clear what to do with the data, but many a times, if you are not provided with a proper documentation of the Video Metadata Container, you will find yourself puzzled and trying to make sense out of 0's and 1's. (This is what makes it difficult).
If you interpret a float for an integer or vice verse, or you misinterpret fixed real number to floating real number, you will have all the data wrong, and not a single clue why this may have happened.

Many a times, the camera manufacturers hide the important metadata in fields that were officially not designed for it. And there are many other similar issues that may arise.

Then there is another problem, to prevent the code from crashing down.
A through testing is crucial, and many a times online support is minimum. I mean we cant add files to test suite for which we do not have the consent of the owner itself (copyright issues).

Last time I tried my best and was roughly able to collect some fifty videos for testing purposes. And being honest, it was not sufficient, as we saw a lot of code breakdown in later tests as well.

I know C++ well, but I lack the knowledge of video metadata and the Exiv2 library. I'm ready to look up the documentation and learn. Can anyone please guide me through this?

Yes, we can.
As Robin said earlier,
[1] Download from SVN (http://www.exiv2.org/download.html)
[2] Compile and run tests suites (video + photographs) just to ensure that everything is ok
[3] There are some examples given in http://www.exiv2.org/doc/examples.html
Go through these, especially http://www.exiv2.org/doc/xmpsample_8cpp-example.html, cause that would be a major task in this project.
[4] It would be really good if you can submit some patches on any open/unresolved issue from http://dev.exiv2.org/projects/exiv2/issues

[5] Other than that, it would be good if you get started with some research on metadata fields as well.
This will help you draft a better proposal.

Hope this will help you get started, else you may ask further questions.

Regards,
Max
