Script offering (2 scripts) - find and clean files that throw EXIF errors in Exiv2 CLI

Added by Steve Wright about 10 years ago

Both these scripts use commands and routines current to BASH 4.0+ and Exiv2 2.1.1 and later. Each has been tested reasonably extensively in Cygwin 1.79 and Ubuntu GNOME.

Here's what each script does.

exivate.sh finds the errors, and creates the text file list of errant files that cleanxiv2err.sh corrects by removing the poorly-written EXIF data from the files.

Hope someone finds them useful.

SJ Wright

exivate.sh.txt (666 Bytes) exivate.sh.txt Find which files in a list throw EXIF errors on the command line.
cleanxiv2err.sh.txt (1.79 KB) cleanxiv2err.sh.txt Correct errors found with 'exivate' script (NB: deletes EXIF data from files).

RE: Script offering (2 scripts) - find and clean files that throw EXIF errors in Exiv2 CLI - Added by Steve Wright about 10 years ago

The oddest thing just happened to me this morning with my original copy of cleanxiv2err.

I got an error that I knew I didn't get the last 6 times I ran it. So I checked it with a "file foo" command. As you can tell by perusing the version I uploaded, it has a crunchbang line in it, so of course that didn't tell me anything about the line-endings. Fortunately Crimson Editor did: it indicated right away that it was still in DOS. I'm presuming that when I saved it, I had already saved something that had to be saved CR/LF (like I'd never heard of dos2unix -- just lazy, I guess!) and forgot to set the prefs back to saving in Unix line-endings. Unlike BBEdit and TextWrangler on the Mac, and now GNOME's gedit application, you can't make those tweaks in a save dialog (how I wish you could: http://forum.emeraldeditor.com/index.php?topic=7881.0 ).

So take care that the script you run is ready in more ways than just having an execute bit. :)

SJ Wright
