Linux GUI for exiv2 available

Added by Anonymous almost 6 years ago

After seeing several comments asking for a Linux exiv2 GUI front-end, I decided to make one for Linux users using Python/Gtk for the most common functions of exiv2. If anyone is interested, it is a Debian source package (.deb size 21K) installable as is, free, and no need to compile. It uses Python 2.7+ and Gtk 2+. Only tested with exiv2 0.23. Will accept reasonable change requests. I have no website so just e-mail me and I will email back the package (exifgui_0.2-1_all.deb)

Hi Robin,

I attach a copy of version 0.2-1 for Debian. I have not tested it with the Ubuntu Unity Desktop nor Kubuntu but it should work fine. Note that most linux distributions only carry the exiv2 0.23. So that is the version I use as a dependency.

This package version is mainly used by people who wish to either rename their files based on dates and time-zones for pictures taken from different devices, and to
view/anonyize the data in the pictures. That was the start of my work after several trips overseas using 4 devices and trying to assemble slideshows in the right order.

I will post it after you let me know what you think needs to be changed if needed. I have a more recent version (0.2-2) that uses more of the exiv2 features and am awaiting comments from the Linuxmint/Ubuntu users trying it.

The attached file is a .deb file (tar/zip format). It contains the python source and XML .glade source. No compiling required.
