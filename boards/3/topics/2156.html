RE: How to get the metadata from the image instead of path? - Added by Andreas Huggel over 6 years ago

Look only at the middle part of the example. The other lines are mostly there to make it a complete program that actually does something when you run it:

Exiv2::Image::AutoPtr image = Exiv2::ImageFactory::open(buf.pData_, buf.size_);
image->readMetadata();
Exiv2::ExifData &exifData = image->exifData();
exifData["Exif.Image.Model"] = "A long model name to make the Exif metadata grow";
image->writeMetadata();

This demonstrates how to operate on an image in memory. How it got there and what you do with it afterwards is up to you.

The key function is Exiv2::ImageFactory::open(), which in this case takes a pointer to the image in memory and its size as parameters.
Subsequently, all Exiv2 calls to deal with the metadata are the same as if you had read the image from a file.

The tricky part to note is that if you modify any metadata and then write it back to a binary representation, the image size may change and it may be relocated in memory.
You'll have to remember to access the updated image via Exiv2::Image::io(), as the original example shows too.
