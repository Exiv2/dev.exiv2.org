We don't support this compiler because I don't have access to such a system. We support GCC, Clang and MSVC. Tooling up to support another compiler is a considerable undertaking. We have a build server which builds and tests trunk (and selected branches) when code is submitted. Supporting additional build systems requires continuous access to the platform.

While it desirable that Exiv2 builds and runs perfectly on any compiler and platform, that goal cannot be reached because implementations of C++ are so variable.
