This will have to be investigated in #992 which is schedule for v0.27.

v0.26 adds a feature Camera overflow file #1034. I've added the code to OlympusMakerNote::print0x0201 (the LensType interpreter) to look in the section [olympus] for a definition of the lens type. However that code isn't called by _A200528.ORF because the ORF decoder does not appear to have found the 0x0201 tag in the makernote.

So there's no improvement in v0.26 in our handling of this file, however progress is being made.
