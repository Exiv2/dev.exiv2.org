h1. Guidance about serialising metadata using MemIo and ExvImage Classes

Please see application samples/toexv.cpp which has three modes of behaviour:

|_. Mode |_. Command syntax |
| Read an image and serialize the metadata to a .exv file | <code>$ bin/toexv http://clanmills.com/Stonehenge.jpg foo.exv</code> |
| Read an image and serialise the metadata into memory | <code>$ bin/toexv http://clanmills.com/Stonehenge.jpg -</code> |
| Read an image, serialise the metadata into memory and deserialise back to an image | <code>$ bin/toexv http://clanmills.com/Stonehenge.jpg +</code> |

*About .exv files*

A .exv file is a JPEG formatted file which has no graphics. It is supported by exiv2 and exiftool. The following is a typical JPEG:<pre>~/gnu/exiv2/ttt $ exiv2 -pS test/data/Reagan.jpg

*About BasicIo and Image Abstract Classes*

Exiv2 has two base classes for IO and Image processing.

*_BasicIo*_ is an abstract class for performing IO. It includes methods to open/read/write/close files. Concrete classes are derived for FileIo which are implemented using the _Standard C_ FILE structure and functions. MemIo stores the data in memory. A series of classes are derived from RemoteIo to perform I/O using remote protocols such as HTTP, FTP and SSH.

*_BasicIo*_ is an abstract class to represent an image. It includes methods to readMetadata and writeMetadata. The metadata is represented by STL structures. The physical files are read and written using classes derived from BasicIo.

*Convert an image to a .exv file*

The essential code to achieve this is:<pre> std::string readPath;
std::string writePath;

Exiv2::Image::AutoPtr readImage = Exiv2::ImageFactory::open(readPath);
assert(readImage.get() != 0);
readImage->readMetadata();

// create a file, copy and write the metadata
Exiv2::Image::AutoPtr writeImage = Exiv2::ImageFactory::create(Exiv2::ImageType::exv,writePath);
writeImage->setMetadata(readImage);
writeImage->writeMetadata();</pre>

*Read an image and serialise it to memory*

<pre> std::string readPath;
std::string writePath;

Exiv2::Image::AutoPtr readImage = Exiv2::ImageFactory::open(readPath);
assert(readImage.get() != 0);
readImage->readMetadata();

// create an in-memory file
Exiv2::BasicIo::AutoPtr memIo (new Exiv2::MemIo());
Exiv2::Image::AutoPtr memImage(new Exiv2::ExvImage(memIo,true));
memImage.setMetadata(*readImage);

// read image metadata into memory
size_t size = memImage->io().size();

*About application toexv*

The basic operation is to extract the metadata from an image and serialise it into a file. However options are provided to selectively define the categories of metadata supported by exiv2. For v0.26, these are Exif, IPTC, XMP, ICC Profiles and JPEG/Comments. <pre>~/gnu/exiv2/ttt $ bin/toexv -h
Reads and writes raw metadata. Use -h option for help.
Usage: toexv [-:iecCahsx] readfile {-|+|writefile}

Options:
-i Read Iptc data from readfile and write to writefile.
-e Read Exif data from readfile and write to writefile.
-c Read Jpeg comment from readfile and write to writefile.
-C Read ICC profile from readfile and write to writefile.
-x Read XMP data from readfile and write to writefile.
-a Read all metadata from readfile and write to writefile.
-s Print size of writefile.
-h Display this help and exit.
~/gnu/exiv2/ttt $ </pre>
