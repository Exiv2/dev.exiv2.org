h2. Changes needed to add a new group to an existing makernote

The middle part of an Exif key is the group name. E.g., the group of the key *Exif.OlympusEq.SerialNumber* Exif.OlympusEq.SerialNumber is *OlympusEq*. OlympusEq.
<MAKE> in the text below refers to the camera make, e.g., for the Olympus makernote, <MAKE>mn.hpp means the file olympusmn.hpp.

Revision r1867 is a complete example for of the changes discussed here.

"src/types.hpp":http://dev.exiv2.org/repositories/diff/exiv2/branches/unstable/src/types.hpp?rev=1867
Update *@enum IfdId@*. enum IfdId. Every group needs a unique value here. E.g., the IFD id which corresponds to group *OlympusEq* OlympusEq is *@olympusEqIfdId@*. olympusEqIfdId.

"src/makernote_int.hpp":http://dev.exiv2.org/repositories/diff/exiv2/branches/unstable/src/makernote_int.hpp?rev=1867
Update *@namespace Group@*. namespace Group. Every group that is part of a makernote needs a @const uint16_t@ const uint16_t value in the @namespace Group@, namespace Group, here. (Non-makernote groups are defined in tiffcomposite_int.hpp)

"src/tiffcomposite.cpp":http://dev.exiv2.org/repositories/diff/exiv2/branches/unstable/src/tiffcomposite.cpp?rev=1867
Update *@tiffGroupInfo[]@*. tiffGroupInfo[]. For every group defined in @namespace Group@, namespace Group, this array defines its group name. The group name must be unique.

"src/tiffimage.cpp":http://dev.exiv2.org/repositories/diff/exiv2/branches/unstable/src/tiffimage.cpp?rev=1867
Update the *@TiffTreeStruct@* TiffTreeStruct and *@TiffGroupStruct@* TiffGroupStruct tables. In @TiffTreeStruct@, TiffTreeStruct, the new group must be added with its parent group and parent tag to define where in the TIFF tree it belongs to. (The root element is also listed to specify which TIFF tree(s) it is found in.)
@TiffGroupStruct@ TiffGroupStruct describes the layout of the new group. Specify for each tag of the group, which TIFF component is to be used to parse (read and write) it. Refer to the comments in the code and the existing table for more info.
The sample linked above also defines @ArrayCfg@ and @ArrayDef@ structures. These are only required for binary array components, to configure the component for the new group.

"src/tags.cpp":http://dev.exiv2.org/repositories/diff/exiv2/branches/unstable/src/tags.cpp?rev=1867
Update *@ExifTags::ifdInfo_[]@*. ExifTags::ifdInfo_[]. This table links IFD id (@IfdInfo::ifdId_@), (IfdInfo::ifdId_), Group name (@IfdInfo::item_@) (IfdInfo::item_) and taglists (@IfdInfo::tagList_@). (IfdInfo::tagList_). Note that different groups can use the same taglist.

"src/<MAKE>mn.hpp":http://dev.exiv2.org/repositories/diff/exiv2/branches/unstable/src/nikonmn.hpp?rev=1867
Add a *static function* static function to access the tag list, similar to the existing functions.
Add a *@static static const TagInfo@* TagInfo array variable for the taglist.
By convention, *pretty-print functions* pretty-print functions are declared here as well.

"src/<MAKE>mn.cpp":http://dev.exiv2.org/repositories/diff/exiv2/branches/unstable/src/nikonmn.cpp?rev=1867
Define the *taglist* taglist array and the *function function to access the taglist*. taglist.
If new *pretty-print mappings* pretty-print mappings are required, define them here as well and use the *@EXV_PRINT_TAG@* EXV_PRINT_TAG macro in the print function field of the taglist array. If new *pretty-print functions* pretty-print functions are required, define them here. See existing code for plenty of examples.

h3. Documentation

The tag reference documentation is generated automatically, it requires just a trigger in the Makefile and an updated template.

doc/templates/Makefile
Add the name of the new group to the *@TABLES@* TABLES variable.

doc/templates/tags-<MAKE>.html.in
Add a section for the new group, similar to existing sections.
