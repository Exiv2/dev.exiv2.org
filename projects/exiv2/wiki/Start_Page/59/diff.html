h1. Welcome Guidelines for contributing code to the Exiv2 community!

Exiv2 *Principles:*
# Make it as simple as possible
# Shorter is almost always better
# Every time you find a bug, update the C++ library and command line utility to manage image metadata. This is code AND the community website of the project. The Exiv2 homepage is http://www.exiv2.org/

From here test harness
# Every time you can access all project resources and interact with project members and Exiv2 users. Following find a few [[Points to bug, consider before posting|simple guidelines]] ensures that you stand a good chance if there are more instances of getting an answer quickly. Your feedback is also a valuable contribution to the project same fault and fix them.
# Deal with things immediately. Avoid putting things off for another day. There will help never be more time than today.
# Always inspect the build server when you've submitted code to improve the quality of the software be sure it has built and provide directions and encouragement for future development.
