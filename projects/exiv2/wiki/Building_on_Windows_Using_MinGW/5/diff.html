h1. Building exiv2 on Windows Using MinGW

# Download "@MinGW@":http://sourceforge.net/projects/mingw/files/Installer/mingw-get-setup.exe/download @MinGW@ from http://sourceforge.net/projects/mingw/files/Installer/mingw-get-setup.exe/download
# While selecting packages, make sure that the selected packages include the ones listed in the file "MinGW_Required_Packages.txt":http://dev.exiv2.org/attachments/658/MinGW_Required_Packages.txt
# pkg-config is not part of standard MinGW installation. Assuming your @MinGW@ installation is in @C:\MinGW@, download "pkg-config":http://ftp.gnome.org/pub/gnome/binaries/win32/dependencies/pkg-config_0.23-3_win32.zip http://ftp.gnome.org/pub/gnome/binaries/win32/dependencies/pkg-config_0.23-3_win32.zip and extract @pkg-config.exe@ executable to @C:\MinGW\bin@. Then download "glib":http://ftp.gnome.org/pub/gnome/binaries/win32/glib/2.28/glib_2.28.1-1_win32.zip http://ftp.gnome.org/pub/gnome/binaries/win32/glib/2.28/glib_2.28.1-1_win32.zip and extract liglib-2.0-0.dll to @C:\MinGW\bin@. Open command prompt and type:
<pre>
cd C:/MinGW/bin
cp libintl-8.dll intl.dll
</pre>
# For building exiv2, open command prompt and type:
<pre>
mkdir C:\gnu
cd C:\gnu
svn checkout svn://dev.exiv2.org/svn/trunk
cd trunk
set PATH=c:\MinGW\bin;c:\MinGW\msys\1.0\bin;C:\MinGW\msys\1.0\local\bin;
bash
PS1="\! \${PWD}> "; #Optional
make config
./configure
make
make install
export PKG_CONFIG_PATH=/usr/local/lib/pkgconfig
make samples
make tests
</pre>
## Troubleshooting: If you get the error...
<pre>
/usr/bin/grep: /home/keith/staged/mingw32/lib/libiconv.la: No such file or directory
/usr/bin/sed: can't read /home/keith/staged/mingw32/lib/libiconv.la: No such file or directory
libtool: link: `/home/keith/staged/mingw32/lib/libiconv.la' is not a valid libtool archive
</pre>
...then open command prompt and type:
<pre>
grep keith C:\MinGW\lib
</pre>
and remove all *.la files containing that word.
Refer: "keith-hard-link-bug":http://sourceforge.net/p/mingw/bugs/2216/ http://sourceforge.net/p/mingw/bugs/2216/
