Points to consider before posting¶

  1. Spend some time trying to figure out the answer to your question before asking for help. Read the available documentation, search this site, the old mailing list and use your favourite search engine to check if your issue has been raised before or your question has been answered already. Experiment with the software or the command line tool.
  2. Decide whether to post to the forums, raise a new issue or add your comments to an existing bug, feature or forum post. If you have found a bug and you're reasonably sure that it really is a new bug, it's best to open an issue for it. If the issue is with a specific image, we'll most likely need the image, please append it to the bug report if possible.
  3. In any case, posting here is better than writing to any project member in private as this allows others to discuss the issue and supply answers which a specific individual may not be able to give. Also, others can later find these answers without having to ask again.
  4. If you'd like to get involved in the development or contribute code, please post your ideas in the forum before sending a patch, unless it addresses a specific open issue.

The bottom line is that if you manage to keep the administrative efforts of the people from whom you're requesting help minimal, you stand a better chance of getting what you are looking for.

There is an extensive guide with tips and explanations on how to get help for Open Source projects in general, written by real hackers. Please note that this document is not part of the Exiv2 project, so don't write to its authors for Exiv2 help.
