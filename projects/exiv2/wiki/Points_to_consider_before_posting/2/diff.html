h1. Points to consider before posting

# Search this site, the "old mailing list":http://uk.groups.yahoo.com/group/exiv2 and use your favourite search engine to check if your issue has been raised before or your question has been answered already.
# Decide whether to post to the forums, raise a new issue or add your comments to an existing bug, feature or forum post. If you have found a bug and you're reasonably sure that it really is a new bug, it's best to open an issue for it. If the issue is with a specific image, we'll most likely need the image, please append it to the bug report if possible.
# In any case, posting here is better than writing to any project member in private as this allows others to discuss the issue and supply answers which the person may not be able to give. Also, other users can later find these answers without having to ask again.
# If you'd like to get involved in the development or contribute code, please post your ideas in the forum before sending a patch, unless it addresses a specific open issue.

There is an extensive "guide":http://www.catb.org/~esr/faqs/smart-questions.html "HOWTO document":http://www.catb.org/~esr/faqs/smart-questions.html with tips and explanations on how to get help for Open Source projects in general, written by real hackers. general. But pleazse note that this document is not part of the Exiv2 project, so please don't write to its authors for Exiv2 help.
