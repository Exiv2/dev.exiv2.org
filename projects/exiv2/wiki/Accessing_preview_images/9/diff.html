h1. Accessing preview images

Since release 0.18, Exiv2 contains For the 0.18 release, Vladimir Nadvornik has contributed functionality to easily access preview images embedded in the metadata (Exif, IPTC or XMP).

RAW images may contain several (I've seen up to three) preview images in addition to the main image in different sizes and usually in JPEG or TIFF format. They range from tiny thumbnails to full-sized JPEG images.

Applications can use the preview list to select which preview image is suitable and work with a preview of the actual picture without having to extract and potentially decode the large main image.

The main API of the new functionality is "@Exiv2::PreviewManager@":http://www.exiv2.org/doc/classExiv2_1_1PreviewManager.html. The interface It is designed to make it transparent to applications where and how exactly the preview images are stored in the metadata. It metadata and has little overhead in order to allow fast access to even the tiniest thumbnail.

This example demonstrates shows how it is used:

Exiv2::PreviewManager loader(*image);
Exiv2::PreviewPropertiesList list = loader.getPreviewProperties();

// Some application logic to select one of the previews from the list
Exiv2::PreviewPropertiesList::const_iterator pos = selectPreview(list);

// Get the selected preview image
Exiv2::PreviewImage preview = loader.getPreviewImage(*pos);
