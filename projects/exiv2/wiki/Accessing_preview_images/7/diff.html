h1. Accessing preview images

For the 0.18 release, Vladimir Nadvornik has contributed functionality to easily access preview images embedded in the metadata (Exif, IPTC or XMP).

RAW images may contain several (I've seen up to three) preview images in addition to the main image in different sizes and usually in JPEG or TIFF format. They range from tiny thumbnails to full-sized JPEG images.

Applications can use the preview list these to select which preview image is suitable and work with very quickly display a thumbnail or lower resolution preview of the actual picture without having to extract (which is typically large and potentially decode the large main image. may require decoding before it can be displayed).

The main API of For the 0.18 release, Vladimir Nadvornik has contributed new Exiv2 functionality is "@Exiv2::PreviewManager@":http://www.exiv2.org/doc/classExiv2_1_1PreviewManager.html. It is designed to make it transparent to applications where and how exactly easily access a list of available preview images as well as the actual preview images are stored in from the metadata. metadata of the image (Exif, IPTC or XMP).

This example shows how it is used:

<pre><code class="c">Exiv2::Image::AutoPtr image = Exiv2::ImageFactory::open(filename);
image->readMetadata();

// Get a list of preview images available in the image. The list is sorted
// by the preview image pixel size, starting with the smallest preview.
Exiv2::PreviewManager loader(*image);
Exiv2::PreviewPropertiesList list = loader.getPreviewProperties();

// Some application logic to select one of the previews from the list
Exiv2::PreviewPropertiesList::const_iterator pos = selectPreview(list);

// Get the selected preview image
Exiv2::PreviewImage preview = loader.getPreviewImage(*pos);

// Write the preview image to a file
std::string previewname = "preview" + pvImg.extension();
std::cout << "Writing preview" << " "
<< preview.width() << "x" << preview.height() << " "
<< "to file" << " " << previewname << std::endl;

long rc = preview.writeFile(pvFile);
