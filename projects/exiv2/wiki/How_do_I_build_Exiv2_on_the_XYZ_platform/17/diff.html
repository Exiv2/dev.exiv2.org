h1. How do I build Exiv2 _on the XYZ platform_?

Exiv2 is developed on GNU/Linux with recent versions of GCC. Written in C++, it uses the standard C++ library, C++ exceptions and a few POSIX and C functions and is fairly portable. The "distribution":http://www.exiv2.org/download.html includes standard build files for UNIX-like systems (including GNU/Linux, Mac OS X and MinGW) as well as Visual Studio project files for Visual C++ .NET 2003 (MSVC 7.1) and later. Additionally, we provide "use at your own risk" support for building with CMake.

For Unix type systems (Mac/Linux/Cygwin/MinGW), the normal build command-set is:
<pre>
$ cd <exiv2-dir>
$ make config
$ ./configure
$ make # make CXXFLAGS=-ggdb for debug
$ sudo make install
*** TAKE CARE: You need the utility pkg-config to build the samples AND the following environment variable ***
$ export PKG_CONFIG_PATH="/usr/local/lib/pkgconfig:/usr/lib/pkgconfig:/opt/local/lib/pkgconfig"
$ make samples
*** TAKE CARE: You must build the samples to run the tests ***
$ make tests
</pre>For CMake on Unix systems (see <exiv2dir>README-CMAKE for details and information about CMake/Visual Studio)
<pre>
$ cd <exiv2-dir>
$ mkdir ../build
$ cd ../build
$ cmake --help # to get a list of "generators" on your machine.
$ cmake -G "Unix Makefiles" ../<exiv2-dir> # -DCMAKE_BUILD_TYPE=Debug for debugging
$ make # builds exiv2 samples
$ make tests
$ sudo make install (if you wish to install)
</pre>For Windows, Visual Studio support provides a robust environment for building static/dynamic/debug/release/32/64 bit builds of the libraries and all sample applications.
Visual Studio support is provided in 2 flavors:
<pre>
msvc64: for 32 bit and 64 bit builds with Visual Studio 2005/8/10 More details: <exiv2-dir>/msvc64/ReadMe.txt
msvc: for 32 bit builds with Visual Studio 2003/5/8. More details: <exiv2-dir>/msvc/README-MSVC.txt
</pre>For Unix/Mac/Linux users of Eclipse or QtCreator or Xcode or other tools

* I've provided notes about building and debugging in Eclipse, QtCreator, and Xcode.
* For dedicated code warriors, I'm confident you can use emacs/vim/gdb and other arcanary.
* Choose your weapon and may the force be with you!
